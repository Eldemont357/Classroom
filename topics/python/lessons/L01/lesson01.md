# Введение

На начальном этапе изучения программирования особенно важно, чтобы каждая конструкция языка была связана не с абстрактным примером, а с конкретной задачей.

Если вводить переменные, условия и операции как отдельные, не связанные между собой элементы, то у обучающихся возникает фрагментарное восприятие языка. Они видят отдельные команды, но не понимают, как из них строится программа.

Поэтому в рамках данного занятия изучение базовых конструкций Python организуется в формате проекта.

В качестве проекта рассматривается разработка текстовой игры. Такой формат позволяет сразу показать, что программа — это не набор случайных строк кода, а система, в которой:

* хранятся данные о состоянии игры;
* принимаются решения;
* изменяются параметры персонажа;
* выбираются дальнейшие действия;
* формируется результат в зависимости от действий игрока.

В ходе занятия будет создана простая текстовая игра, в которой игрок управляет персонажем, принимает решения, изменяет параметры своего состояния и приходит либо к победе, либо к поражению.

# Постановка задачи

Необходимо разработать простую текстовую игру.

Игрок должен:

* ввести имя персонажа;
* получить начальные характеристики;
* выбрать одно из нескольких действий;
* изменить состояние персонажа в зависимости от выбора;
* пройти несколько игровых ситуаций;
* получить итог игры.

Таким образом, уже в первом проекте появляется важное для программирования понятие **состояния системы**.

Состояние игры — это совокупность значений переменных в конкретный момент времени.

Например:

* имя героя;
* количество здоровья;
* количество золота;
* наличие предмета;
* факт победы или поражения.

Именно через такие данные программа “помнит”, что уже произошло и что должно произойти дальше.

# Ход работы

## 1. Общая идея проекта

Перед написанием кода необходимо понять, как программа будет работать.

В простейшем виде игра должна выполнять следующие шаги:

1. вывести вступительный текст;
2. запросить имя игрока;
3. создать начальные значения игровых параметров;
4. предложить выбор действия;
5. обработать выбор;
6. изменить состояние игры;
7. вывести результат;
8. проверить, завершена ли игра.

Даже такая простая схема уже показывает, что программа — это не один фрагмент кода, а последовательность взаимосвязанных этапов.

---

## 2. Переменные как основа игрового состояния

Любая игра должна где-то хранить свои данные.

Для этого используются переменные.

Переменная — это имя, связанное с некоторым значением, которое может изменяться в процессе выполнения программы.

В проекте игры переменные нужны для хранения состояния персонажа.

Пример начальных переменных:

```python
player_name = "Герой"
health = 100
gold = 10
has_sword = False
```

Здесь:

* `player_name` хранит имя игрока;
* `health` хранит уровень здоровья;
* `gold` хранит количество золота;
* `has_sword` хранит логическое значение, указывающее, есть ли у персонажа меч.

Следует обратить внимание на то, что переменные могут хранить разные типы данных:

* строки;
* числа;
* логические значения.

Без переменных игра не может существовать как программа, потому что ей негде фиксировать изменения.

Если здоровье игрока уменьшилось, программа должна где-то это сохранить.
Если игрок получил предмет, программа должна об этом помнить.
Если игрок победил, это тоже должно быть отражено в данных.

---

## 3. Как выполняется программа

Очень важно объяснить обучающимся, что программа на Python выполняется последовательно, сверху вниз.

Например:

```python
health = 100
health = health - 20
print(health)
```

Порядок выполнения будет следующим:

1. создаётся переменная `health` со значением 100;
2. вычисляется выражение `health - 20`;
3. результат записывается обратно в `health`;
4. значение выводится на экран.

После выполнения программы переменная `health` будет содержать 80.

Это особенно важно для понимания игровой логики.
Сильная группа должна понимать, что изменение состояния — это не “магия игры”, а результат последовательного пересчёта значений переменных.

---

## 4. Ввод и вывод данных

Поскольку игра является интерактивной программой, необходимо обеспечить обмен информацией между программой и игроком.

Для вывода используется функция `print()`.

```python
print("Добро пожаловать в игру")
```

Для ввода используется функция `input()`.

```python
player_name = input("Введите имя героя: ")
```

Следует отдельно подчеркнуть, что `input()` всегда возвращает строку.

Это означает, что при вводе числовых данных необходимо преобразование типа.

Например:

```python
choice = int(input("Введите номер действия: "))
```

Если не выполнить преобразование, дальнейшие сравнения и вычисления могут работать неверно.

---

## 5. Операции в проекте игры

В игре постоянно выполняются операции над данными:

* уменьшение здоровья;
* увеличение золота;
* проверка количества ресурсов;
* вычисление итогового результата.

Примеры:

```python
health = health - 15
gold = gold + 20
damage = 10 + 5
```

Здесь используются стандартные арифметические операции.

Особое значение имеет обновление переменной через саму себя:

```python
health = health - 20
```

Именно такая форма записи используется в играх постоянно, так как игровые параметры почти всегда изменяются во времени.

---

## 6. Условия как механизм выбора сценария

Игра невозможна без выбора.

Если игрок идёт в лес, развивается один сценарий.
Если игрок идёт в пещеру, развивается другой сценарий.
Если у игрока мало здоровья, он проигрывает.
Если игрок нашёл сокровище, он получает награду.

Все такие ситуации реализуются с помощью условного оператора.

Простейшая форма:

```python
if health <= 0:
    print("Игра окончена")
```

Более сложный пример:

```python
if choice == 1:
    print("Вы вошли в лес")
elif choice == 2:
    print("Вы вошли в пещеру")
else:
    print("Некорректный выбор")
```

Условный оператор позволяет программе изменить своё поведение в зависимости от текущего состояния и действий пользователя.

Именно поэтому условия являются центральным элементом любой интерактивной программы.

## 7. Проект игры: логика первого варианта

В рамках первого занятия создаётся игра следующего типа:

Игрок оказывается перед развилкой и должен принять несколько решений:

* куда пойти;
* сражаться или отступить;
* открыть сундук или проигнорировать его.

Каждое решение влияет на параметры персонажа.

Введём следующую игровую модель:

* у героя есть имя;
* у героя есть здоровье;
* у героя есть золото;
* у героя может появиться меч;
* если здоровье становится меньше либо равно нулю, игра заканчивается поражением;
* если герой находит сокровище и выживает, игра заканчивается победой.

## 8. Полный код первого проекта

```python
print("Добро пожаловать в текстовую игру.")
print("Сегодня вам предстоит пройти небольшое приключение.")

player_name = input("Введите имя героя: ")

health = 100
gold = 0
has_sword = False

print()
print(f"Герой {player_name} начинает путь.")
print(f"Здоровье: {health}")
print(f"Золото: {gold}")
print()

print("Перед вами развилка.")
print("1 — пойти в лес")
print("2 — пойти в пещеру")

choice1 = input("Ваш выбор: ")

if choice1 == "1":
    print()
    print("Вы входите в лес и встречаете волка.")
    print("1 — сражаться")
    print("2 — убежать")

    choice2 = input("Ваш выбор: ")

    if choice2 == "1":
        print()
        print("Вы победили волка, но получили ранения.")
        health = health - 30
        gold = gold + 20
        print(f"Текущее здоровье: {health}")
        print(f"Текущее золото: {gold}")
    elif choice2 == "2":
        print()
        print("Вы сумели убежать, но потеряли часть припасов.")
        health = health - 10
        print(f"Текущее здоровье: {health}")
    else:
        print()
        print("Некорректный выбор. Герой растерялся и потерял время.")
        health = health - 5
        print(f"Текущее здоровье: {health}")

elif choice1 == "2":
    print()
    print("Вы входите в пещеру и находите сундук.")
    print("1 — открыть сундук")
    print("2 — пройти мимо")

    choice2 = input("Ваш выбор: ")

    if choice2 == "1":
        print()
        print("В сундуке оказался меч.")
        has_sword = True
        print("Теперь у вас есть оружие.")
    elif choice2 == "2":
        print()
        print("Вы прошли мимо сундука и ничего не получили.")
    else:
        print()
        print("Некорректный выбор. Вы споткнулись о камень.")
        health = health - 5
        print(f"Текущее здоровье: {health}")

else:
    print()
    print("Некорректный выбор направления. Герой потерялся.")
    health = 0

print()
print("Следующий этап приключения.")

if health <= 0:
    print("Герой погиб. Игра окончена.")
else:
    print("На пути появляется страж сокровищ.")
    print("1 — сражаться")
    print("2 — отступить")

    choice3 = input("Ваш выбор: ")

    if choice3 == "1":
        if has_sword:
            print()
            print("Благодаря мечу вы победили стража.")
            gold = gold + 100
        else:
            print()
            print("Без оружия победа далась тяжело.")
            health = health - 50
            gold = gold + 30
    elif choice3 == "2":
        print()
        print("Вы отступили и сохранили жизнь, но не получили сокровища.")
    else:
        print()
        print("Некорректный выбор. Страж атакует первым.")
        health = health - 20

print()
print("Итоги игры:")
print(f"Герой: {player_name}")
print(f"Здоровье: {health}")
print(f"Золото: {gold}")
print(f"Меч: {has_sword}")

if health <= 0:
    print("Результат: поражение.")
elif gold >= 100:
    print("Результат: полная победа.")
elif gold > 0:
    print("Результат: частичный успех.")
else:
    print("Результат: герой выжил, но не добился цели.")
```

## 9. Разбор структуры проекта

После показа кода необходимо не просто запустить программу, а разобрать её архитектуру.

### 9.1. Блок инициализации

```python

gold = 0
```

Этот блок отвечает за начальное состояние игры.

Фактически здесь создаётся модель персонажа.

---

### 9.2. Блок первого выбора


Здесь запускается первая точка ветвления сценария.

Программа ожидает решение игрока и в зависимости от значения переменной `choice1` переходит к одной из ветвей.


Здесь наиболее наглядно видно назначение переменных:

### 9.4. Блок проверки завершения игры


Здесь реализуется итоговая интерпретация состояния игры.

Это очень важный момент:

## 11. Типичные ошибки при разработке проекта

### 11.1. Ошибка сравнения с числом вместо строки

```python
choice = input("Ваш выбор: ")

if choice == 1:
    print("Вы выбрали первый вариант")
```

Такой код не сработает корректно, потому что `input()` возвращает строку.

---

### 11.2. Ошибка пропуска обновления состояния

Студент может написать сюжетный текст, но забыть изменить переменную.

Например, в тексте герой получил урон, но `health` не изменилось.
В результате сценарий и данные расходятся.

---

### 11.3. Ошибка с некорректной логикой финала

Если сначала проверить золото, а потом здоровье, можно случайно объявить победителем героя, который уже погиб.

Следовательно, при проектировании программы необходимо учитывать приоритет условий.

---

### 11.4. Ошибка случайной перезаписи

```python
gold = 0
gold = 100
gold = 5
```

Если студент не отслеживает последовательность выполнения, итоговое значение может оказаться не тем, которое ожидается.
